## 项目主要需求

### 一、项目概述

**项目名称**：校园培训管理系统（CampusHub）
**项目短名**：wjecampus
**项目类型**：全栈 Web 应用
**前端端口**：38203
**后端端口**：38303

本系统面向高校培训机构，为管理员、教师、学生三类角色提供课程管理、选课、作业考试发布与提交、成绩管理、考勤管理、操作审计等全流程功能。系统要求前后端完全分离，代码严禁合并职责到单一文件，每个核心实体必须贯穿 DB -> Model -> Service -> Controller -> API -> Store -> 组件 全链路。

| 前端框架 | Vue 3 + TypeScript |

### 三、核心实体定义

所有实体贯穿全栈：数据库表 -> SQLAlchemy Model -> Service -> Controller(API Router) -> 前端 API 封装 -> Pinia Store -> Vue 组件。每个实体独立文件，严禁合并。

#### 3.1 课程 Course

**数据库字段**：
- `id` (UUID, 主键)
- `name` (VARCHAR(200), 课程名称, NOT NULL)
- `code` (VARCHAR(50), 课程编号, UNIQUE, NOT NULL)
- `teacher_id` (UUID, 外键关联 User 表, NOT NULL)
- `description` (TEXT, 课程简介)
- `max_students` (INTEGER, 最大选课人数, 默认 60)
- `semester` (VARCHAR(20), 学期, 如 "2026-Spring")
- `status` (ENUM: DRAFT / PUBLISHED / IN_PROGRESS / COMPLETED / ARCHIVED, 默认 DRAFT)
- `created_at`, `updated_at` (TIMESTAMP)

**关联关系**：
- 一门课程有多个班级（通过课程-班级关联表）
- 一门课程有多个作业/考试（Assignment.course_id）
- 一门课程有多个考勤记录（Attendance.course_id）
- 一门课程有多名学生选课（通过选课关联表 enrollment）

**业务动作**：开课（教师创建）、选课（学生选课）、退课（学生退课）、结课（教师结课并归档）

#### 3.2 学生 Student

**数据库字段**：
- `id` (UUID, 主键)
- `student_no` (VARCHAR(30), 学号, UNIQUE, NOT NULL)
- `name` (VARCHAR(100), 姓名, NOT NULL)
- `grade` (VARCHAR(10), 年级, 如 "2024")
- `class_name` (VARCHAR(50), 班级)
- `phone` (VARCHAR(20), 联系电话)
- `email` (VARCHAR(120), 邮箱)
- `enroll_date` (DATE, 入学日期)
- `user_id` (UUID, 外键关联 User 表, UNIQUE)
- `created_at`, `updated_at` (TIMESTAMP)

**关联关系**：
- 一名学生可选多门课程（通过 enrollment 关联表）
- 一名学生有多条考勤记录
- 一名学生有多条作业提交记录

**业务动作**：注册、选课、退课、查看成绩

#### 3.3 作业/考试 Assignment

**数据库字段**：
- `id` (UUID, 主键)
- `course_id` (UUID, 外键关联 Course, NOT NULL)
- `title` (VARCHAR(200), 标题, NOT NULL)
- `type` (ENUM: HOMEWORK / EXAM / QUIZ / PROJECT, NOT NULL)
- `description` (TEXT, 详细说明)
- `deadline` (TIMESTAMP, 截止时间, NOT NULL)
- `total_score` (DECIMAL(5,1), 满分, NOT NULL)
- `weight` (DECIMAL(3,2), 权重占比, 如 0.20 表示 20%)
- `status` (ENUM: DRAFT / PUBLISHED / CLOSED, 默认 DRAFT)
- `created_at`, `updated_at` (TIMESTAMP)

**关联关系**：
- 属于一门课程（Assignment.course_id -> Course.id）
- 一个作业有多份学生提交（Submission.assignment_id）

**业务动作**：发布（教师发布）、提交（学生提交）、批改（教师打分批改）、发布成绩（教师发布批改结果）

#### 3.4 考勤 Attendance

**数据库字段**：
- `id` (UUID, 主键)
- `course_id` (UUID, 外键关联 Course, NOT NULL)
- `student_id` (UUID, 外键关联 Student, NOT NULL)
- `date` (DATE, 考勤日期, NOT NULL)
- `status` (ENUM: PENDING / PRESENT / ABSENT / LATE / LEAVE, 默认 PENDING)
- `remark` (VARCHAR(200), 备注)
- `created_at`, `updated_at` (TIMESTAMP)

**关联关系**：
- 多对多：一门课程 + 一名学生 在某一天产生一条考勤记录
- UNIQUE 约束 (course_id, student_id, date)

**业务动作**：教师点名、学生查看出勤记录、教师修改考勤状态

### 八、后端中间件设计

#### 8.1 认证中间件 `auth_middleware.py`

- 从请求 Header 提取 JWT Token
- 验证 Token 有效性，解析用户信息
- 将用户信息注入到请求上下文中
- 白名单路由（如 `/api/auth/login`）跳过认证

#### 8.2 授权中间件 `authorization_middleware.py`

- 根据路由配置的权限要求，检查当前用户角色是否满足
- 支持角色 + 资源级别的权限检查（如教师只能操作自己的课程）
- 权限不足时返回 403

#### 8.3 审计中间件 `audit_middleware.py`

- 拦截所有写操作（POST/PUT/PATCH/DELETE）
- 记录操作人、操作类型、操作目标、请求参数
- 对于修改操作，记录修改前后的数据差异
- 异步写入 `audit_logs` 表，不阻塞主请求

#### 8.4 全局异常处理中间件 `error_handler.py`

- 捕获所有未处理异常，统一返回标准错误响应格式
- 区分业务异常（400/404/403）和系统异常（500）
- 系统异常记录详细日志（含堆栈），但对外只返回通用错误信息
- 标准错误响应格式：`{ "code": number, "message": string, "detail": string | null }`

### 九、前端错误处理

#### 9.1 请求拦截器

在 Axios 请求拦截器中统一处理：
- 请求拦截：自动附加 JWT Token 到 Header
- 响应拦截：根据 HTTP 状态码统一展示错误提示
  - 401：清除本地 Token，跳转登录页
  - 403：ElMessage 提示"无权限执行此操作"
  - 404：ElMessage 提示"请求的资源不存在"
  - 422：ElMessage 展示表单校验错误详情
  - 500：ElMessage 提示"服务器内部错误，请稍后重试"

#### 9.2 路由守卫

- 未登录用户访问受保护页面 -> 重定向到 `/login`
- 已登录用户访问无权限页面 -> 重定向到 403 页面或默认首页
- 路由 meta 中配置 `requiredRole` 字段

### 十一、代码规范与架构约束

**严禁合并职责到单一文件**。以下为强制约束：

1. **一个实体 = 一组文件**：每个实体必须有独立的 model、service、controller、schema、API 封装、Store、类型声明文件
2. **中间件独立文件**：每个中间件一个文件，不合并到 app.py 或 main.py
3. **枚举独立文件**：后端 `enums.py`，前端 `enums.ts` + `permissions.ts`，严禁在业务文件中定义枚举
4. **组件独立文件**：每个共享组件一个 `.vue` 文件，不合并到一个大组件中
5. **Hooks 独立文件**：每个 Hook 一个 `.ts` 文件
6. **路由配置独立文件**：`router/index.ts` 单独维护
7. **API 封装按实体分文件**：`api/courses.ts`、`api/students.ts` 等，不合并为一个 `api.ts`

### 十二、数据库初始化

- `database/init.sql`：创建数据库和扩展
- `database/migrations/`：Alembic 迁移脚本
- `database/seeds/`：种子数据（包含管理员账号、示例教师、示例学生、示例课程）

## 部署要求

项目必须支持 Docker Compose 一键部署，且在 **任意目录名（包括中文目录名）** 下都能正常启动。

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: wjecampus`
- 所有服务使用 `container_name` 命名容器，带 `${COMPOSE_PROJECT_NAME:-wjecampus}` 前缀
- 数据库数据使用命名卷持久化
- 前端通过 Nginx 反向代理后端 API（调用 `/api`，由 Nginx 转发到 `backend:38303`）
- 数据库配置 `healthcheck`，后端 `depends_on` 数据库健康
- 后端配置 `healthcheck`，前端 `depends_on` 后端健康
- 所有可变配置通过环境变量注入

需要提供：docker-compose.yml, .env, .env.example, frontend/Dockerfile, backend/Dockerfile, frontend/nginx.conf

### docker-compose.yml 示例结构

```yaml
name: wjecampus

services:
  db:
    image: postgres:16-alpine
    container_name: ${COMPOSE_PROJECT_NAME:-wjecampus}-db
    environment:
      POSTGRES_DB: ${POSTGRES_DB}
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER}"]
      interval: 5s
      timeout: 5s
      retries: 5

  backend:
    build: ./backend
    container_name: ${COMPOSE_PROJECT_NAME:-wjecampus}-backend
    environment:
      DATABASE_URL: postgresql://${POSTGRES_USER}:${POSTGRES_PASSWORD}@db:5432/${POSTGRES_DB}
      JWT_SECRET: ${JWT_SECRET}
    depends_on:
      db:
        condition: service_healthy
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:38303/api/health"]
      interval: 10s
      timeout: 5s
      retries: 3

  frontend:
    build: ./frontend
    container_name: ${COMPOSE_PROJECT_NAME:-wjecampus}-frontend
    ports:
      - "${FRONTEND_PORT:-38203}:80"
    depends_on:
      backend:
        condition: service_healthy

volumes:
  pgdata:
```
